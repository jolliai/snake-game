## Summary

Customer request: support boards that have **holes in the middle** — interior wall/obstacle cells inside the playable region that the snake must navigate around (colliding with one is fatal, like hitting the board edge).

## Context

This extends the existing **Iron Snake mode**, which already carves the board into an irregular shape using a flat mask (`Uint8Array`, `1` = playable, `0` = off-board/wall) generated in `src/iron-snake.ts`. Today those shapes are solid, simply-connected blobs (a union of overlapping rectangles pruned to the largest connected component). This feature would punch interior holes into that region so the interior is no longer solid.

## Scope / considerations

* Board generation: introduce interior holes while guaranteeing the playable region stays a single 4-connected component (snake can always reach all food) and respects the existing min-width (no 1–2 cell corridors) constraint.
* Collision: moving onto a hole cell should be a death, surfaced with a clear loss reason (consistent with the existing death-freeze behavior).
* Food placement: never spawn food on a hole.
* Rendering: holes should read clearly as gaps/walls in both the isometric and overhead views, and animate sensibly through the level-to-level morph transition (<issue id="d1331647-debe-4e11-b06f-0fc777db7609" href="https://linear.app/joesthewizardworkspace/issue/JOE-5/create-smooth-transition-from-level-to-level-in-iron-snake-mode">JOE-5</issue>).
* Bots: pathfinding must treat holes as obstacles.
